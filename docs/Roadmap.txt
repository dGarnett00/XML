4) Technical Design Doc (Concise Version)
4.1 Objectives
Offline XML editor with advanced bulk ops
Handles very large XML files efficiently
Intuitive UI with power features
Local versioning + undo/redo
4.2 Architecture Overview
Layered system:

UI Layer

Table, Tree, Raw XML views
Command palette
Timeline & diff viewer
Application Layer

Document controller
Search/filter engine
Bulk edit rules engine
Schema validation
Data Layer

XML parsing + streaming
Indexed node store
Local database (SQLite)
Automation Layer

Macro engine
Scripting (Lua/JS)
Offline assistant
4.3 XML Engine
Streaming parser to load huge XML without blocking UI
Builds a node index and lightweight tree
Lazy‑loads subtree content on expand
Saves XML via incremental serializer
4.4 Editing & Rules Engine
Rule pipeline:
Filter nodes
Preview changes
Apply transactional edits
Index refresh
Bulk operations are atomic for crash safety
4.5 Storage & Versioning
Each save produces a snapshot diff
Timeline replays via diff chain
Autosave every 5 minutes (configurable)
File locking prevents corruption
4.6 Performance Targets
Load 2M+ nodes in under 10 seconds
Search 100k nodes < 200ms
Bulk edit 10k nodes < 2s
UI stays responsive at all times
4.7 Offline Smart Assist (Optional)
Local inference engine (small model)
Translates natural language → rule expressions
Works without internet
4.8 Extensibility
Plugin API for new views and exporters
Add new schema types
Optional Pro modules
4.9 Security
No telemetry, no network
Fully local file operations
Encrypted project store optional
