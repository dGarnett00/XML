Phase 0 — Foundation (Weeks 1–2)
Goal: Nail the architecture for performance, offline safety, and future extensibility.

Deliverables

Offline-first architecture (no external calls)
Local state engine with autosave + versioning
Data layer optimized for large XML (millions of nodes)
Secure storage for presets, templates, and backups
Plugin system scaffolding
Key Decisions

UI framework: Desktop native (Tauri/Qt/.NET/Swift)
XML engine: streaming parser + in-memory indexed tree
Storage: local database (SQLite) for indexes and history
Phase 1 — Core Editor (Weeks 3–6)
Goal: Build the essential XML editing environment.

Core Features

Split view: Raw XML ⇄ Smart table ⇄ Tree view
Instant search across elements, attributes, and text nodes
Fast filtering (by tag, attribute, regex, ranges)
Inline validation with live error highlighting
Undo/redo across sessions
Crash recovery and auto-save timeline
Intuitive UX

Drag & drop XML files
One-click “Enable Editing”
Sticky header for column sorting
Clean separation of content, attributes, and tags

Core Principles
Offline Always
No network needed. Everything happens locally.

Blazing Fast
Handles millions of nodes with smooth UI.

Intuitive

State‑of‑the‑Art
Uses modern UX patterns, AI‑style queries, modular tooling.

Futuristic
Timeline editing, semantic intelligence, live previews.

Suggested Tech Stack (Offline & Modern)
UI: Tauri (Rust + Web UI) or .NET MAUI
XML engine: Rust (quick-xml) or C# (System.Xml + custom index)
Storage: SQLite for metadata + history snapshots
Plugins: WebAssembly or Lua
